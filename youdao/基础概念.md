### ORM、DAO、DO、BO、DTO、VO、AO、PO、UID 名词意义

![领域模型中的实体类](https://s6.51cto.com/oss/202103/08/eececd500ff18fa256175a1cdd24b19a.jpg)

- `ORM（Object Relational Mapping）`：对象关系模型。是一种将数据库实体映射成对象的概念，可以理解为：一个对象就是数据库中的一个表，通过访问对象的方式来访问数据表的信息
- `DO（Data Object）`：与数据库表结构一一对应，通过 DAO 层向上传输资源对象
- `DAO(Data access object)`： 数据访问对象。通过统一的 API 对象来操作不同类型的数据库，使得上层代码不需要考虑数据引擎或表的关系
- `DTO（ Data Transfer Object）`：数据传输对象，Service 或 Manager 向外传输的对象
- `BO(Business Object) `：业务对象，由 Service 层输出的封装业务逻辑的对象，把业务逻辑封装为一个对象（这是一个复杂对象，里面可能包含多个类）
- `AO（ Application Object）`：应用对象。 在Web层与Service层之间抽象的复用对象模型，极为贴近展示层，复用度不高
- `VO（ View Object）`：显示层对象，通常是Web向模板渲染引擎层传输的对象

## 网络技术名词

### DNS 记录类型

参考：[DNS 记录类型](https://www.cnblogs.com/bluestorm/p/10345334.html)

- 域名解析（DNS 解析）：就是域名到IP地址的转换过程，相比 IP，使用域名标识站点更加简单好记
- [SSL 证书：什么是 SSL 证书 – 定义和解释](https://www.kaspersky.com.cn/resource-center/definitions/what-is-a-ssl-certificate)-为了提高数据的安全性，防止数据在传输过程中被窃取
- CNNIC 隐私保护：在腾讯云/阿里云上买域名的时候会提示这个，针对 国家顶级域名（.CN、.中国）的所有者的个人信息不被公开。例如 [whois](https://www.whois.com/whois) 查询个人信息将不会被显示出来。
- TTL（Time To Live）：生存周期，表示解析记录在 DNS 服务器中缓存的时间，一般是 3600 秒
- A 记录：A-address，A 记录就是将一个域名解析到一个特定的 IP-v4 地址，在公网域名解析经常遇到
- AAAA记录：是将一个域名解析到一个特定的 IP-v6 地址，目前IP-v6 还未普及，见的不多
- NS 记录：Name Server 记录是指域名服务器记录，用于指定该域名是由哪个 DNS 服务器来进行解析
- MX 记录：Mail Exchanger 记录，记录邮件服务器的解析记录（查看 MX 记录：`nslookup -q=mx <domain>` 或 `dig mx <domain>`）
- CNAME：又称“别名记录”，即：将A/B/C等多个主机名(即：域名)都指向一个别名 D 主机名。这样有什么好处呢？当A/B/C服务域名的DNS需要改动时，A/B/C指向D不需要调整，只需要改动D-->DNS即可


CNAME 最常见应用场景是在：CDN 中的使用（参考：[CDN 工作原理](https://support.huaweicloud.com/productdesc-cdn/cdn_01_0109.html)），用户只需要将域名和 CDN 提供的 CNAME 做指定，至于具体的 IP 链路选择由 CDN 服务器自行选择更换。

通过 `CNAME` 将 Domain 和 IP 进行了一层关系解耦，有利于避免 Domain、IP 相互修改时产生的互相影响。
