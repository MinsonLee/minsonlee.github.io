## 所有用户输入都是邪恶的

如果你了解过`SQL注入`，那么你一定也听过这句话：“**所有用户输入都是邪恶的**”。这句话不是要跟大家争论“人性本恶/本善论”，但却给所有互联网从业人员-产品、开发、测试一个提醒，要对用户有一定防备之心。

**敏感信息的展示不要使用规则的数字**，这大概也是开发之间的一个共识，譬如：电商系统中的优惠券自增ID、用户系统中的用户自增ID...因为这种展现方式一旦被人找到漏洞，很容易被人“拖库”（即：被人把整个数据库的信息给扒取下来），造成整个平台的数据泄露。

## 爬取付费网站下的所有文章

今天来跟大家演示一下这个案例，昨天我在一个网站上查看文章，看了几篇之后提示我要积分才能看。

![积分不足](ACE90081D41946049864843D93D60294)

当我鼠标滑过地址栏准备关闭时，无意间瞟到了 URL 的链接有点意思。

我发现这个网站的正常文章是 `view/1.html`、`view/2.html`，而付费是 `view/vip_100.html` 这样的方式。普通文章和 VIP 文章的数字虽然不是连续的，但我想 VIP 的文章大概率也是走的另一套连续数字编码，连续观察了几篇 VIP 文章的地址，发现确实是这样。恰好之前看过几眼 Python，就想着试试能否把这个文章爬取一下试试。

```
# Python Version : 3.8.5
import urllib.request as req
baseurl = 'http://c.biancheng.net/view/'
basef='/mnt/d/clan/'
for i in range(0, 5000):
    try:
        url=baseurl+str(i+1)+'.html'
        webpage = req.urlopen(url) # 根据超链访问链接的网页
        data = webpage.read() # 读取超链网页数据
        data = data.decode('utf-8') # byte类型解码为字符串
        f=basef+'/'+str(i+1)+'.html'
        txtf=open(f,'w',encoding='utf-8')
        txtf.write(data)
        txtf.close()
    except:
        print('失败爬取html:'+str(i+1)+'\n')
```

短短几行代码将网站上的所有文章给扒取了下来，我只尝试扒取了一部分 2524 篇就结束了，结果如截图

![爬取结果](5F4919E747914FB785DB2FAF9CE9E939)

那么现在有个问题：怎么知道每一个网页的文章标题呢？利用 Shell 的三大杀器（grep/sed/awk）中的 `grep -R` 将 `<title> </title>` 标签匹配出来，结果如图 

![匹配title](4B1D964807D54C389FCA6BAF13963F4F)

然后利用 `awk` 方式对所有文章进行批量自动重命名

![image](B5A93D4B74CA49109646593F84D10A1B)

```sh
grep -REo "<title>.*<\/title>" [0-9]*.html | awk -F":<title>|</title>" '{print "mv \""$1"\" \""$2"-"$1"\""}' | bash
```

美中不足之处，由于部分标题中含有目录符号 `/` 不符合文件命令规范，所以提示重命名失败，但这类占文件少数，不过可以通过 `sed` 对特殊符号进行替换，然后再自动执行

![image](467F49C36F8C4985AAE5C3BEDC1C7FB1)

-----

```sh
# 将 html 内容中的 vip_xxxx.html 替换为 xxx.html

grep -REo "vip_[0-9]+\.html" | cut -d':' -f1 | uniq | sort -n | xargs -L 1 -I {} sed -i 's/vip_\([0-9]\+\)\.html/\1\.html/g' {}
```

```sh
# 添加域名
grep -R -E '(src|href)=\"/[^v\"]' |  cut -d':' -f1 | uniq | sort -n | xargs -L 1 -I {} sed -i 's/\(\(src\|href\)="\)\/\([^v"]\)/\1http:\/\/data.biancheng.net\/\3/g' {}
```
