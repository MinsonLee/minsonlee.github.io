## 什么是接口？

接口的意义在于“约束”。

## Golang 中接口与 PHP 接口的对比


```php
<?php
/**
 * @desc 接口：约束类的行为
 */
 interface Sky {
    public function fly();
 }
 
/**
* @desc 抽象类：代码复用 
*/
abstract class Bird  implements Sky {
     
    abstract public function getValue();
    
    public function fly() {
        return $this->getValue() . '-鸟类-在天空飞翔';
    }
}
 
abstract class Plane implements Sky {
    abstract public function getCompany();
     
    public function fly() {
        return $this->getCompany() . '系列的飞机，在天空飞翔';
    }
 }

/**
 * 具体类
 */
class Boyin extends Plane {
    public function getCompany() {
        return '波音';
    }
}

class Eagle extends Bird {
    public function getValue() {
        return '苍鹰';
    }
}

class Dog {}

/**
 * 测试类
 */
class TestClass {
    // 1. 判断对象是否属于接口：<obj> instanceof <Interface>
    public function test ($obj) {
        if($obj instanceof Sky) {
            echo $obj->fly() . PHP_EOL;    
        }
        else {
            echo '什么玩意儿？不属于空军范畴' . PHP_EOL;
        }
    }

    // 2. 判断对象是否属于接口：参数形式，若不是接口则异常
    public function Test2(Sky $sky) {
        echo $sky->fly() . PHP_EOL;  
    }
}

$eagle = new Eagle();
$boyin = new Boyin();
$dog = new Dog();

$test = new TestClass();
$test->test($boyin);
$test->test($dog);
$test->test2($eagle);
```
