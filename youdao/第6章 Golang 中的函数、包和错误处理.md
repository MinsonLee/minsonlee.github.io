## 函数和包
### 什么是函数？
完成某一个功能的程序指令(语句)的集合，称为：函数！程序中使用函数，提高了：代码复用性、可读性、可维护性！

若按照函数的定义主体来划分，`Golang` 中的函数可以分为：**系统内置函数**、**自定义函数**

定义函数的基本语法如下：
```golang
func 函数名 (形参列表:Type var,...) (返回值列表:Type var, ...) {
        语句集合
        return
}

或

func 函数名 (形参列表:Type var,...) (返回类型列表:Type1, Type2, ...) {
        语句集合
        return var1, var2, ...
}
```

### 什么是包？
`包`是具有所定义格式的**文件(函数集合)**和**目录(对文件进行归类)**的集合！通俗易懂的说：**包**就是创建不同的**文件夹**来存放**程序文件**！

在`Golang`中，**每一个文件都属于一个包**，也就是说：**`Go`是包的形式来进行管理文件和项目目录结构的**。

包的作用（类似于`PHP`中的**命名空间**）：
1. **区分**相同名字的函数、变量等标识符，避免冲突
2. 对程序文件进行分文别类，**提高代码的维护性**
3. 控制函数、变量等访问方位，即：**控制作用域**

定义一个文件所在的包，其语法如下：
```golang
package 包名
```

在一个文件中引入其他的包，其语法如下：
```golang
import [别名] "包的路径"

当需要引入多个包时，官方推荐书写如下：

import (
    [别名] "包的路径"
)
```
> 默认包的路径是相对于：`$GOPATH` 去进行相对路径寻址的，在`Go.1.14`版本及以后，官方推荐使用`Go Modules`方式进行包管理

### 包的注意事项
1. `Go`中文件的包名通常和**文件所在的文件夹名一致**，一般为**小写字母**
2. 当一个文件要使用其它的包函数或变量时，需要先引入对应的包
3. 声明包名的语句`package 包名`必须在文件第一行，然后是 `import` 函数
4. 在`import 包的路径`时，包的路径是相对于：`$GOPATH/src/` 下进行相对路径寻址
5. 包中的函数通过**首字母大小写方式控制访问权限**：函数名首字母大写表示所有包能访问；小写表示只能在当前包中访问
6. 访问其他包中函数，语法：**包名.函数名**
7. 同一个包下，不能有相同的函数名/全局变量名（**同一命名空间下，函数和常量不能重复定义使用**）
8. 通过`go build`编译器会将用户自定义的包编码为一个库文件，放到`$GOPATH/pkg`下，可以直接引入使用


### 函数调用机制
```golang
package main
import "fmt"
func test(n1 int) {
    n1 = n1 + 1
    fmt.Println("test() n1=", n1) // 打印 11
}

func getSum(n1 int, n2 int) int {
    sum := n1 + n2
    fmt.Println("getSum sum=", sum) // 30
    return sum
}

func main() {
    n1 := 10
    test(n1)
    fmt.Println("main() n1=", n1) // 打印 10
    
    n2 := 20
    sum := getSum(n1, n2)
    fmt.Println("main sum=", sum) // 30
}
```
![操作系统内存逻辑分区](8FA60629E1094C6EA477F4DF78D35C64)
> 上述的分区，并非操作系统中真实的分区，而是开发者们对操作系统对内存分配的逻辑分区方式

1. 在`Golang`中，**基本数据类型**==一般来说==分配到栈区
2. 引用数据类型一般来说分配到堆区，编译器存在一个逃逸分析
3. 函数的生命周期：调用一个函数时，操作系统会给该函数分配一个新的栈空间，调用完毕后，Golang 中会自动销毁该函数对应的栈空间。编译器会通过自身的处理将新开辟的空间和其它栈区区分开来
4. 函数的作用域：每个函数对应的栈中，数据空间是独立的，不会混淆
5. 当函数有`return`语句时，就是将结果返回给调用者，即：谁调用，就返回给谁

### 函数返回值：`return`语句
```golang
func 函数名(形参列表) (返回值类型列表) {
    // 语句块
    return 返回值列表
}
或
func 函数名(形参列表) (返回值列表) {
    // 语句块
    return
}
```
1. Golang 函数支持返回**多个返回值**，这一点是其它编程语言没有的
2. 返回多个值时，在接收时若希望忽略某个返回值，可以使用匿名变量`_`表示占位忽略
3. 若返回值只有一个，`(返回值类型列表)` 可以不写 `()`

### 函数的递归调用
递归调用（Recursive call）：一个函数**在函数体内又调用了本身**

```golang
// test(4) 将会输出什么？
func test(n int) {
	if n > 2 {
		n--
		test(n)
	}
	fmt.Println("n=", n)
}

// test2(4) 将会输出什么？
func test2(n int) {
	if n > 2 {
		n--
		test2(n)
	} else {
		fmt.Println("n=", n)
	}
}
```

#### 递归的注意事项
1. 执行调用一次函数时候，就会开辟一个新的独立的空间(栈空间)
2. 函数的局部变量只在当前的栈空间内又效
3. 递归必须要有出口(称之为：**尾递归**)，且每次调用后必须向尾递归逼近，否则就成无限递归了
4. 当一个函数执行完毕或遇到`return`就会返回，遵守谁调用，就将结果返回给谁，函数执行完毕或返回时
5. 该函数本身的栈空间也会被系统(编译器)销毁

### 函数的注意事项
1. 函数的形参列表可以是多个，**返回值列表也可以是多个**
2. 形参列表和返回值列表的数据类型可以使值类型和引用类型
3. 函数的命名遵循标识符命名规范，首字母不能是大写，首字母大写表示函数可以被本包和其它包文件使用，类似`public`；首字母小写，只能被本包文件使用，其它包文件不能使用，类似`private`
4. 函数中的变量是局部的，函数外不生效
5. **基础数据类型**和**数组**默认都是**值传递**，即进行值拷贝。在函数内修改，不会影响原来的值
> [Go语言传值和深浅复制问题](http://kchu.me/2016/03/27/Go%E8%AF%AD%E8%A8%80%E4%BC%A0%E5%80%BC%E5%92%8C%E6%B7%B1%E6%B5%85%E5%A4%8D%E5%88%B6%E9%97%AE%E9%A2%98/)
6. 如果希望函数的变量能修改函数外的变量(默认以值传递的方式的数据类型)，可以传入变量的地址`&`，函数内以指针方式操作变量（实际：传入变量的地址就是将值传递变为引用传递）
7. `Go` 函数不支持函数重载【方法重载是面向对象多态特性的表现】
8. 在`Go`中，函数也是一种数据类型，可将其赋值给一个变量，则该变量就是一个函数的变量了，可通过变量对函数进行调用（就像给函数起了一个别名）
9. **函数既然可以是一种数据类型，那么`Go`中，函数也可以作为形参，并且调用**
```golang
func test(n1 int, n2 int) int {
    return n1 + n2
}

func myFunc(funcVar func(int, int) int, num1 int, num2 int) int {
    return funcVar(num1, num2)
}
```
