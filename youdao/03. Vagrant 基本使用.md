# Vagrant 入门指南

该指南使用 Vagrant 和 VirtualBox 搭配演示，因为：VirtualBox 免费开源、跨平台，因此 Vagrant 默认也是和 VirtualBox 搭配使用，但 Vagrant 是可以和其他的虚拟机软件一起使用。请提前安装好 Vagrant 和 VirtualBox

- Vagrant ：[https://www.vagrantup.com/downloads.html](https://www.vagrantup.com/downloads.html)
- VirtualBox ：[https://www.virtualbox.org/wiki/Downloads](https://www.virtualbox.org/wiki/Downloads)

> 注意：
> - Vagrant 官方写到 Vagrant 支持 VirtualBox `4.x` `5.x` `6.x` 的版本
> - 由于 VirtualBox 某些 `5.x` 的版本是存在问题的，例如：共享文件存在问题，`VBoxGuestAdditions` 安装存在问题等导致在 Vagrant 启动的时候 `vagrant up` 的时候报错导致启动失败。部分问题 Vagrant 提供了 插件进行了修复，貌似不是全部，毕竟不是自己的锅
>
> 安装建议：
> - 如果是自己学习 Vagrant 建议两个软件都使用最新的版本
> - 若其中一个软件你安装了旧版本可能出现上述问题，可以问一下跟你安装相同版本的同事或网友他们使用的另一个软件版本

```
# 初始化一个 Vagrant Box hashicorp/precise64
$ vagrant init hashicorp/precise64
# 去 Vagrant Boxes 市场拉取下载指定 Box【这是一个漫长的过程】，并启动该 Box
$ vagrant up
```
通过上述两个命令，你就可以在将 `Ubuntu 12.04 LTS 64-bit` 在 VirtualBox 中完整的跑来了。你也可以通过 `vagrant ssh` 命令以 SSH 进入该虚拟机，当你用完了该虚拟机的时候可以使用 `vagrant destroy` 销毁掉。

## 项目建立
> 原文：https://www.vagrantup.com/intro/getting-started/project_setup.html

配置任何 Vagrant 项目的第一步是创建 Vagrantfile 文件. 该文件的目的有以下两点：

1. 标记该文件所在的目录为整个 Vagrant 项目的根目录。Vagrant 中的许多配置选项都与该目录相关
2. 描述 Vagrant 中的虚拟机和它需要分配的系统资源（例如：`主机名`、`CPU 个数`等信息），以及安装了那些软件服务和如何访问这些软件服务

可以使用`Vagrant init`内建命令初始化一个目录，安装完 vagrant  服务后，在命令行中进行如下操作：
```shell
# 创建一个目录 vagrant_getting_started 作为此次整个 vagrant 项目的根目录 
# 该操作不是必须，你也可以在现有的存在目录中进行初始化的操作
$ mkdir vagrant_getting_started

# 进入将要作为 vagrant 根目录的文件夹
$ cd vagrant_getting_started

# 初始化该目录
$ vagrant init
```

通过上述的操作，将会在你当前的文件夹中生成一个 Vagrantfile 文件，该文件里面有一些命令和示例，教你如何进行配置。

如果在你的 Vagrant 项目使用了版本控制，你需要将 Vagrantfile 文件提交到版本控制中去。这样做的好处是：所有人项目的成员都不需要重复的进行前期的初始化工作。

> 我想使用 Vagrant 的目的大部分都是为了统一工作环境，因此提交 vagrant 到版本控制中（本质即：统一使用同一份 Vagrantfile ）的好处也就不用多说了。

### 初始化命令：vagrant init

**完整用法：** `vagrnat init [options] [name [url]]`
```shell
$ vagrant.exe init my-box https://boxes.my.com/my-box.box
```

- **选项**

|选项 | 是否有短选项 | 短选项 | 描述|
|-----|--------------|--------|-----|
|--box-version Version | 否 | —— |  指定添加 box 的版本|
|--force | 是 | -f | 创建一个 Vagrantfile 文件，若已存在强制创建|
|--minimal | 是 | -m | 创建一个最小的 Vagrantfile (没有注释/示例) |
|--output FILE| 否 | —— | 自定义 Vagrantfile 的文件名 |
|--template FILE| 否 | —— | 通过指定的 ERB 模板文件生成 Vagrantfile |

- **name** : box 的名称
```shell
# 设置初始化 box 名称，即：在virtualbox GUI 管理界面看到的虚拟机名称
# 你可以在 https://vagrantcloud.com/search 中搜索对应的box，并下载至本地
config.vm.box = "my-box"
```

- **url**：描述/指定了 box 的来源
```shell
# 1. 描述 config.vm.box 的 box 来源
# 2. 若在执行 vagrant box add xxx/xxx.box 时候，xxx/xxx.box 没有在当前系统指定位置找到，则会拉取该 box 链接的指定的 box
config.vm.box_url = "https://boxes.my.com/my-box.box"
```



## Boxes 管理
> 原文：https://www.vagrantup.com/intro/getting-started/boxes.html

Vagrant 使用一个基础映像快速克隆来构建一个新的虚拟机，而不是从头开始构建一个新的虚拟机(从头构建一个虚拟机是一个缓慢并且乏味的过程)。这些基本映像（注意:映像和镜像是不同的）在 vagrant 被称为 `boxes`。在初始化创建 `Vagrantfile` 后，为 Vagrant 环境指定 box 是第一步。

### 安装 Box
Vgrant 通过 `vagrant box add` 添加 box，在 Vagrant 下所有 box 都有一个唯一的名称或id，方便我们在 Vagrant 的环境中重复的使用它们，如果你已经添加了该 Box `hashicorp/precise64` 则不用再重复添加：

```shell
$ vagrant box add hashicorp/precise64
```

上述命令会从 [HashiCorp's Vagrant Cloud box catalog](https://vagrantcloud.com/boxes/search) 这个平台上去下载一个名叫 `hashicorp/precise64` 的 box。当然你也可以通过 `本地 box 文件`、`自定义 box 文件的 URL` 来下载指定的 box。

Box 是为全局所有用户添加的。每一个项目都根据基础映像文件（xxx.box文件）来克隆一个 box，并且永远不会更改到原来的基础映像文件。这意味着：如果你有两个项目都是使用 `hashicorp/precise64` 来创建的 box，`vagrant box add` 命令只需要执行一次，但是一个用户在一个项目中加了一个文件是不会影响到另一个项目的。

通过上述命令我们可以意识到：应用市场的 box 是有命名空间的。它包含两部分信息：`用户名`、`box 名称`。
在上述的案例中，用户名是 `hashicorp` ，box 名称是 `precise64`。你也可以通过 `本地 box 文件`、`自定义 box 文件的 URL` 来下载指定的 box，但是官方不提供指南介绍了。


### 配置 box

Box 添加了之后，我们需要打开 Vagrantfile 来配置 box，将其作为项目的基础。打开一份最简单的 Vagrantfile：

```shell
Vagrant.configure("2") do |config|
  config.vm.box = "hashicorp/precise64"
end
```
- vagrant 的所有配置都写在 `Vagrant.configure("2") do |config|` .. `end`中间
- `Vagrant.configure("2") do |config|` 中的 "2" 表示配置的版本标识（Vagrant 支持更改为更加旧的版本标识，因为该配置的版本是向下兼容的）。不建议修改，除非你真的弄明白了你修改版本标识是在做什么
- `config.vm.box = "hashicorp/precise64"` 用于配置 box 的命名空间。命令中的`hashicorp/precise64`必须要和曾经`vagrant box add ADRESS`匹配上。如果这个 box 没有被 `add` 过，那么你在启动的时候 Vagrant 会自动的去下载

若你添加的 box 是有版本控制的，你还可以指定要使用的 box 版本（当然这个配置项是可选的），例如：
```shell
Vagrant.configure("2") do |config|
  config.vm.box = "hashicorp/precise64"
  # 指定使用的 box 的版本
  config.vm.box_version = "1.1.0"
end
```

当你本地没有 box 文件，你可以通过 `config.vm.box_url` 自定义 box 的具体路径，例如：
```shell
Vagrant.configure("2") do |config|
  config.vm.box = "hashicorp/precise64"
  config.vm.box_url = "https://vagrantcloud.com/hashicorp/precise64"
end
```
> 启动 box 的时候，box 的寻找优先级如下：
> `本地添加的 box` 》 `config.vm.box_url 指定的资源路径` 》 [`Vagrant 官方 box 应用市场：HashiCorp's Vagrant Cloud box catalog`](https://vagrantcloud.com/boxes/search)

## 启动/SSH
> 原文：https://www.vagrantup.com/intro/getting-started/up.html

启动 Vagrant 环境非常简单，进入 Vagrant 项目所在的根目录，执行`up`命令：

```sheel
$ vagrant up
```

通过以上命令，你就已经在后台开启了一台没有 UI 操作界面的虚拟机了，启动后你可以通过`SSH`方式进入虚拟机，例如：

```shell
$ vagrant ssh
```

这个命令将建立一个 SSH 会话，让你可以直接操作虚拟机。虽然该项目下的虚拟机是临时独立的，但是你仍然要小心注意 `rm -rf /` 命令，因为 Vagrant 中同时可能挂载了你主机上的目录作为共享目录，这个共享目录的设置在 Vagrantfile 中设置， `rm -rf /` 同时也会删除共享目录中所有的文件。

花点时间思考一下刚才发生的事情：仅通过在命令行输入一行配置和一行命令，我们就得到了一个功能齐全的SSH可访问虚拟机。SSH 会话可以通过 `CTRL+D` 组合快捷键终止连接。
```shell
vagrant@precise64:~$ logout
Connection to 127.0.0.1 closed.
```

当你使用完了该虚拟机，你可以在你的主机上使用 `vagrant destroy` 命令，这样 Vagrant 将会终止虚拟机对你主机的所有的资源占用。
> 注意：`vagrant box remove` 命令不是真正的删除你已经下载的 box 文件，如果你想真正的删除虚拟机下的所有文件，请使用 `vagrant destroy`命令

1. 什么是虚拟化？
>指通过虚拟化技术将一台计算机虚拟为多台逻辑计算机。在一台计算机上同时运行多个逻辑计算机，每个逻辑计算机可运行不同的操作系统，并且应用程序都可以在相互独立的空间内运行而互不影响，从而显著提高计算机的工作效率。==> 虚拟化是一个概念

2. 什么是虚拟化技术？
>是一种资源管理技术，是将计算机的各种实体资源，如服务器、网络、内存及存储等，予以抽象、转换后呈现出来，打破实体结构间的不可切割的障碍，使用户可以比原本的组态更好的方式来应用这些资源！
>
>==> 虚拟化技术是虚拟化概念实现的解决方案。典型的虚拟化技术代表有：
> 1. 在硬件抽象层基于 Hypervisors(虚拟机管理程序) 方式来衍生、运行、销毁各种不同操作系统的虚拟机：
>       - Xen，KVM，VMWare，QUEM
>       - Hyper-V(微软提出的一种系统管理程序虚拟化技术)
> 2. 基于 cgroup 和 namespaces 容器(Linux Container，缩写：LXC)

3. 什么是虚拟机?
> 通过虚拟化技术模拟的具有完整硬件系统功能的、运行在一个完全隔离环境中的完整计算机系统。==> 虚拟机是虚拟化概念的产物！
