- 机器语言：0101 这样的二进制数据，并不适合人类进行阅读
- 汇编语言：可读性更好的机器语言，每一条指令都可以直接通过汇编器翻译成一条机器码
- 高级语言：如 C/JAVA/GO/PHP/JS 这类利于用户编写的的开发语言，支持多种编程范式

## 编译器

目的：编译原理不是屠龙技，而是为了**理解编程语言的设计、实现原理**，从而更好的掌握编程语言，排查问题。

**编译器就是一个二进制可执行程序**，作用就是把**语言代码文本翻译为机器平台能执行的二进制代码**，机器平台二进制可执行文件可以和汇编语言指令一一对应，每个汇编指令都有其对应的二进制格式的机器码。

## C 语言是如何实现自举的呢？

- [编程语言的自举](https://www.cnblogs.com/lidyan/p/6727184.html)：用语言自身实现自己的编译器。如：C语言 用 C语言 实现了`GNU C`编译器（即：`GCC`）。
- [操作系统的自举](https://blog.csdn.net/qq_41613281/article/details/104955543)：计算机必须具备自举能力将自己所有启动所必备的元件激活，以便能完成加载操作系统这一目的，然后再由操作系统承担起那些单靠自举代码无法完成的更复杂的任务。
    - 操作系统自举只有两个功能：加电自检和磁盘引导
    - 加电自检：计算机自己通过电路自检，确定所有必备的元件是否能正确工作（不能工作就会听到我们电脑故障经常听到的“嘟嘟”的声音）
    - 磁盘引导：查找装有操作系统的磁盘驱动器，将用户选择的操作系统从磁盘加载到存储器中

**任何语言的第一个编译器一定是非自身语言实现的**。如：C语言的第一个编译器就是用汇编语言写的（暂且称为 V1 编译器），然后再用C语言写一个编译器，利用 V1 编译器来编译我们写的C语言就能得到一个**可执行的、用C语言写的C语言编译器**二进制程序。后续的C语言代码就可以用这个C语言实现的编译器编译C语言代码。


## 编译过程

任何一个编译器其编译过程都是如下：**词法分析-->语法分析-->语义分析-->生成中间代码-->优化-->生成目标代码**

![编译过程](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/编译过程.png)

编译器前端：如何将代码转为语法正确的目标代码；

编译器后端：给你一个计算机，如何让正确的目标代码能正确、高效的运行

### 编译器前端

#### 词法分析（Lexical Analysis）

将代码中的“词法记号（Token）”，如：关键字、标识符、操作符等通过“**正则表达式**”结合“有限自动机算法”识别出来进行**词法切分**。（[Lex-词法分析生成工具](https://www.bwangel.me/2019/12/15/flex/)）

![词法分析-lex](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/lex.png)


#### 语法分析（（Syntactic Analysis, or Parsing））

对词法分析得到的结果进行识别，构造出一个**树状结构**的程序语法结构，一个程序就是一棵“**抽象语法树（Abstract Syntax Tree，AST）**”。

即：将切分得到的词语，根据编程语言自己的规则分文别类得到结构。如：汉语是主谓宾语法结构、日语是主宾谓语法结构。

![抽象语法树-AST](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/ast.png)

对于脚本语言（解释器）来说，执行脚本语言就是用递归下降算法（一种自顶向下的算法）遍历其 AST 树的一个过程。

**语法分析**是整个编译过程中，程序员可能最需要关注的。

#### 语义分析（Semantic Analysis）

让计算机理解代码的真实意图，结合上下文将一些模棱两可的地方去掉，让计算机能依据这些信息升恒目标代码。

如：我喜欢你，这个“我”是谁？“你”是谁？语义分析的工作就是将“我、你”替换为正确的人名。


### 编译器后端

总的来说，编译器后端要解决的问题是：现在给你一台计算机，你怎么生成一个可以运行的程序，然后还能让这个程序在计算机上正确和高效地运行？

![程序运行简易模型](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/20220430213345.png)

![程序的运行环境](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/20220430215533.png)

编译器最终的目的：“生成目标代码”，如：C 语言目标是直接运行在计算机上，那这个目标代码就是汇编代码；Java 代码运行与 Java虚拟机 上，这个目标代码就是 JVM字节码。

汇编代码其实并不难，但是由于其过于简单且不太安全，因此汇编代码常用于处理很小、很单一的问题。如果要用汇编代码处理复杂的问题，那么虽然运行效率会很高，但编写效率会很低下。

因此：复杂的问题，我们都是用高级语言进行编写，然后由高级语言的编译器自动翻译为对应的繁琐的汇编代码。

**中间代码（Intermediate Representation，IR）机制**的发明：因为不同的CPU架构有不同的汇编指令集，为了提高代码的复用效率，我们发明了中间代码机制。

优化分为：**“独立于机器的优化”和“依赖于机器的优化”**

- 独立于机器的优化： 基于 IR 进行，通过对代码的分析，用更加高效的代码替代原来的代码。如：
    - SQL 的编译器会自动的优化我们的 WHERE 条件保证我们能尽可能使用它认为是最优化的索引；
    - 编译器会将我们的一些没有使用到的变量、方法...剔除掉，消除冗余的逻辑
- 依赖于机器的优化：依赖于硬件的特征，使代码能更好的利用计算机的特性。如现代计算机的并行计算能力、多层次的内存结构、寄存器、高速缓存等。

一般对于我们来说，重点关注的也是：CPU、内存。而 CPU 内部结构很多，重点关注的又只是：寄存器、高速缓存。

- 寄存器：是 CPU 指令临时数据存储的地方，寄存器的速度极快，因此：能用寄存器就别用内存
- 高速缓存：为了弥补 CPU 处理速度和内存访问之间的差距而产生。指令在读取内存是会将与当前处理数据相关的一组数据都读进高速缓存内，然后寄存器优先从高速缓存中读取数据。好比：外卖小哥不会每送一笔订单就在顾客和门店间来回跑动，而是相邻的一批订单一起配送。

写程序时，尽量把某个操作所需的数据都放在内存中的连续区域中，不要零零散散地到处放，这样有利于充分利用高速缓存。**这种优化思路，叫做数据的局部性。**

## 一个C语言程序的编译过程？

利用 [GCC](http://c.biancheng.net/view/7930.html) 编译一个 `*.c` 格式的C语言文件，其流程如下：

![C语言编译过程](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/20220501100355.png)

- 预处理（cpp）：`gcc -E hello.c -o hello.i`
- 编译： `gcc -S hello.i -o hello.s`
- 生成目标文件：`gcc -c hello.s -o hello.o` 或 `as hello.s -o hello.o`
- 链接器：`gcc hello.o -o hello`

```c
#include <stdio.h>
int main(int argc, char* argv[]) {
    printf("Hello %s!\n", "Minson");
    return 0;
}
```

通过 `gcc -S -O2 hello.c -o hello.s` 生成汇编代码（`-O2` 告诉编译器进行2级优化）

```sh
        .file   "hello.c"
        .text
        .section        .rodata.str1.1,"aMS",@progbits,1
.LC0:
        .string "Minson"
.LC1:
        .string "Hello %s!\n"
        .section        .text.startup,"ax",@progbits
        .p2align 4,,15
        .globl  main  ## -- Begin function main
        .type   main, @function
main:
.LFB23:
        .cfi_startproc
        leaq    .LC0(%rip), %rdx
        leaq    .LC1(%rip), %rsi
        subq    $8, %rsp
        .cfi_def_cfa_offset 16
        movl    $1, %edi
        xorl    %eax, %eax
        call    __printf_chk@PLT
        xorl    %eax, %eax
        addq    $8, %rsp
        .cfi_def_cfa_offset 8
        ret
        .cfi_endproc  ## -- End function
.LFE23:
        .size   main, .-main
        .ident  "GCC: (Ubuntu 7.5.0-3ubuntu1~18.04) 7.5.0"
        .section        .note.GNU-stack,"",@progbits
```

## 汇编

汇编语法特别简单，但它需要跟硬件（CPU 和 内存）打交道，而不同的处理器架构（x86_64、ARM、Power）其支持的指令集都不同，也就意味着：**同样一个功能的汇编程序在不同的处理器架构中是不同的**。

1. **数据传输类**：将数据从一个位置复制到另一个位置（如:①. 内存-->寄存器; ②. 寄存器-->内存; ③. 寄存器-->寄存器;）
2. **算术与逻辑运行类**：加减乘除、与或运行、左移、右移
3. **控制类**：比较两个值，然后跳转到某个位置（所谓的`while/for/switch...case` 不过就是 `if+goto` 的包装）


汇编语言的组成元素

- **指令（instruction）**：直接由 CPU 进行处理的命令（格式：**助记符 [操作数]** 多个操作数用逗号分隔）
- 伪指令：以`.`开头，末尾没有冒号`: ` ，辅助性指令，是写给汇编器看的（不同汇编器的伪指令集不一样
- 标签：以冒号 `:` 结尾，用于对伪指令生成的数据或指令做标记
- 注释：以 `#` 号开头，与 C 语言中注释语句一样

汇编指令助记符

| 助记符 | 指令种类 | 用途 | 示例 |
| --- | --- | --- | --- |
| MOVQ | 传送 | 数据传送 |  `MOVQ48，AX` 表示把`48`传送`AX`中 |
| LEAQ | 传送 | 地址传送 | `LEAQ AX，BX`表示把`AX`有效地址传送到`BX`中 |
| CALL | 转移 | 调用函数 | `CALL runtime.closechan(c *hchan)`表示通过`closechan`函数的内存地址发起调用 |
| PCDATA | 表格 | PCDATA和FUNCDATA的数据一般是由编译器自动生成的，手工编写并不现实 | 不重要，先忽略 |
| FUNCDATA | 表格 | PCDATA和FUNCDATA的数据一般是由编译器自动生成的，手工编写并不现实 | 不重要，先忽略 |
