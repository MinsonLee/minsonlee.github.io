《Wireshark 网络分析就这么简单》的开篇中有这么一道题：
两台服务器 A 和服务器 B 的网络配置如下，B 的子网掩码本应该是 `255.255.255.0`，被不小心配成了 `255.255.255.224`。它们还能正常通信吗？

选项 | 服务器 A | 服务器 B
---- | -------- | --------
IP | 192.160.26.129 | 192.160.26.3
Subnet mask | 255.255.255.0 | 255.255.255.224
Default gateway | 192.160.26.2 | 192.160.26.2
DNS Server | 127.0.0.1 | 127.0.0.1

> - [子网掩码不同是否可以通信？](https://www.cnblogs.com/hang1368/p/12957804.html)
> - [彻底搞懂系列之：ARP协议](https://zhuanlan.zhihu.com/p/370507243)

通信流程如下：

1、服务器A ping 服务器B：

- 因为服务器 A 根据自己的子网掩码，计算出 B 属于同一个子网，同一个子网通信不需要默认网关的转发，因此：服务A -->ping-->服务B
- 服务器 B 回报给 服务器 A，此次服务器 B 根据自己的子网掩码，计算出 A 属于不同子网，跨子网通信需要默认网关的转发。而要和默认网关通信，就需要获得其 MAC 地址。因此：服务器B --> 回报 --> 默认网关 --> 服务器 A


2、服务器 B ping 服务器 A

- 因为 B 根据自己的子网掩码，计算出 A 属于不同子网，跨子网通信需要默认网关的转发。而要和默认网关通信，就需要获得其 MAC 地址。
- B 发出 ping 包，指定 Destination IP 为 A ，即 192.168.26.129 。但 Destination MAC 却是默认网关的 Mac 地址，这表明 B 希望默认网关把包转发给 A 。
