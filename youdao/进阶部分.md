
## 1.1 容器化的几种架构方式

- Docker 容器引擎
- Docker Compose ：Docker 容器管理工具（只能管理当前主机的 Docker）
- Docker Swarm：Docker 集群管理工具（[基于Docker Swarm集群的调度策略优化算法](http://www.c-s-a.org.cn/html/2018/9/6544.html)：Random(随机), Spread (扩散)和 Binpack (装箱)）
    - spread: 默认策略，尽量均匀分布，找容器数少的结点调度
    - binpack: 和 spread 相反，尽量把一个结点占满再用其他结点
    - random: 随机，实施简单, 但经常容易导致节点CPU过载和内存不足

> - Spread 和 Binpack 调度首先根据申请容器的配置信息计算出 CPU 和内存利用率, 将其结果求和得到节点权重
> - 采用Spread策略, 虽然能够减少因节点故障而损坏容器的数量, 但是这种策略过多的占用了服务器资源(例如内存资源). 
> - Binpack的优点是能将更多的容器运行在较少的节点上, 但是会造成节点负载过重.
> - Random策略的优点是实施简单, 但经常容易导致节点CPU过载和内存不足.
