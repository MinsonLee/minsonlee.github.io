## 容器
### 什么是容器？
- 一种虚拟化方案：基于共享的宿主机操作系统之上，划分独立的、可插拔式的运行空间
- 直接运行在操作系统内核之上的虚拟化技术 ==> 容器虚拟化技术也称"操作系统级虚拟化技术"
- **只能运行相同或相似的内核的操作系统** ==> 相比于虚拟机的劣势
- ==依赖于 Linux 内核体系：`Kernel Namespace`(内核命名空间) 和 `Cgroups（Control Group）`(控制组)、`Union File System`(联合文件系统)==


### Linux 容器其技术 VS 虚拟机

![容器和虚拟机的对比](39A27B0F4B934A4C9B6C3ED97F4758A7)

1. 磁盘占用空间更少
    - 虚拟机：应用+其依赖的库+完整的操作系统
    - 容器：应用+其依赖的库
2. 资源损耗
    - 虚拟机：对硬件和CPU/内存等系统资源进行模拟("基于虚拟化硬件仿真机制")，对系统资源损耗较大，对系统资源利用率低 ==> 和主机高度隔离
    - 容器：共享主机操作系统，由主机直接进行资源分配使用，因此不存在虚拟系统资源的损耗，极大的提升了系统资源的利用率 ==> 和主机的隔离相对较差

### 既然容器的好处如此大，为什么直到 Docker 的出现，容器技术才真正火起来呢？
> https://www.lagou.com/lgeduarticle/8732.html

**容器的痛点在什么地方呢？**

容器技术依赖 Linux 的很多内核特性：内核命名空间(`Kernel Namespace`)、控制组(`Cgroup-Control Group`)、联合文件系统(`Union File System`)。因此我们说容器的时候，实际更多是指：`LXC-Linux Container`(`Linux` 容器)。

因此限制了容器技术只能建立在`Linux`操作系统之上，而`Linux`的学习成本相较于与其他操作系统来说是复杂，因此导致容器不宜安装、不方便统一管理、不方便实现自动化。

而以上的问题，随着 2013 年`Docker`的出现，问题得到了解决：

1. Docker 让容器技术使用起来更加方便了：
    - 让项目交付过程更加职责分明
        - 开发者：关注应用的开发与应用的运行情况
        - 运维：关注容器的管理与维护，如：配置镜像；统一管理、分发；上线时直接部署验收完毕的镜像
        - 测试人员：专注测试工作，不再因为环境差异而浪费时间
    - 方便-轻量、可移植：`Docker`没出现之前，使用`LXC`需要过硬的`Linux`技能，阻碍了容器技术的推广，而`Docker`屏蔽了复杂的操作系统底层依赖技术，使得广大的开发人员可以使用`Docker`方便快捷的构建一个容器。
2. 社区活跃，技术支持雄厚
    - `Docker` 与其他容器领域的巨擘进行了合作，包括 `Canonical`、谷歌、红帽和 `Parallels`，共同开发其关键的开源组件 `libcontainer`
    - `Docker Hub` 上更是有着许许多多的开源人员源源不断的贡献优秀的镜像
3. 面向服务的架构发展：随着当下软件服务的迅猛发展，分布式服务架构越来越被提倡。
    - `Docker` 推荐：==单个容器只运行一个应用程序或进程(独立意味着更稳定)==，这样就形成了一个分布式的应用程序模型。
    > 在这种模型下，应用程序或服务都可以表示为一系列内部互联的容器，从而使分布式部署应用程序，扩展或调试应用程序都变得非常简单，同时也提高了程序的内省性

> 参考： http://dockone.io/article/396


## Docker
### 什么是 Docker？
Docker 是一个能够把开发的应用程序自动部署到容器的开源`**容器引擎**，**Docker本身并不是容器，它只是一个软件**。

### Docker 的目标？
提供一个**轻量、快速的环境**，能够运行开发者的程序，并**方便高效地**将程序从开发者的笔记本**部署**到测试环境，然后再部署到生产环境。
> Docker是革命性的，它重新定义了软件开发、测试、交付和部署的流程。我们交付的东西不再只是代码、配置文件、数据库定义等，而是整个应用程序运行环境：**`OS`+各种中间件、类库+应用程序代码**。

==`Docker` 表面是改变了运维的方式，实质上改变了交付流程！==


### Docker 的使用场景
1. 使用 `Docker` 快速搭建开发、测试、生产服务环境
2. 创建隔离的运行环境
3. 搭建测试环境【因为创建非常简单】
4. 构建度用户的平台即服务（PaaS）基础设施
5. 提供软件即服务（SaaS）应用程序
6. 高性能、超大规模的宿主机部署

### Docker 的组成
![Docker 架构图](C37DD9CA342048BCBF86E2BA89DC04F9)

1. Docker 客户端和服务器
> Docker 是一个客户端/服务器（C/S）架构的程序
- Docker 客户端只需向 Docker 服务器或守护进程(Docker Daemon)发出请求，服务器或守护进程将完成所有工作并返回结果
- Docker 守护进程有时也称为 Docker 引擎
- Docker 提供了一个命令行工具 docker 以及一整套 API 来与守护进程交互
- Docker 客户端和 Docker Daemon 可以是在同一台宿主机上，也可以不在同一台宿主机上

2. Docker 镜像
> 镜像是构建 Dokcer 世界的基石！

- 用户基于镜像来构建、运行自己的容器：镜像是Docker生命周期中的“构建”部分
- 镜像是基于联合（Union）文件系统的一种层式的结构，由一系列指令一步一步构建出来
    - 添加一个文件
    - 执行一个命令
    - 打开一个端口
- 镜像的体积很小，非常便携，易于分享、存储和更新（也可以把镜像当做容器的“源代码”）
- Docker Image 是一个层叠的只读文件系统
    - bootfs 【引导文件系统-Docker 用户几乎永远不会和引导文件系统有交互，当容器启动完毕后会被卸载】
    - rootfs【root文件系统-可以是一种或多种的操作系统，如：Ubuntu、CentOS（大约有100M左右）、Alpine-只有5-10M左右】--传统的Linux引导过程中最先会以只读状态被载入，当引导完毕后再切换为读写模式，而Docker中rootfs永远只会以只读方式被加载 ==> 基础镜像
    - 联合加载（Union mount）-一次同时加载多个文件系统

3. Registry
> 用于保存/管理用户构建的镜像【镜像仓库】
- 公共的 Registry：
    -  Docker Hub - Docker 公司运营
- 私有的 Registry【用户自行搭建】

> K8S 是基于容器的集群管理平台，它的全称，是 Kubernetes！( Registry 是 Docker 的仓库，K8S 相当于 Registry 的仓库)
> - 参考：http://docs.kubernetes.org.cn/251.html

4. Docker 容器
Docker 可以方便、快速的帮用户构建和部署容器，用户只需要把自己的应用程序或服务打包放进容器即可！
> - 容器是基于镜像启动起来的，容器中可以运行一个或多个进程
> - 镜像是 Docker 生命周期中的构建或打包阶段，而容器则是启动或执行阶段

> 思考：Docker 镜像是一个层叠的只读文件系统，那么 Docker 容器是怎样通过镜像来启动运行的呢？
>
> 当 Docker 容器运行时，Docker 会在镜像的最顶层加载一个读写文件系统（即：可写文件层），我们在 Docker 中运行的程序就是在这层中运行执行的，当 Docker 第一次启动时候，初写的读写层是空的；当下层的文件系统发生变化时，所有的变化都会被复制到读写层中（即：写时复制-copy on write 机制）
> 
>参考：《 Docker 镜像之进阶篇》：https://www.cnblogs.com/sparkdev/p/9092082.html

1. 什么是虚拟化？
>指通过虚拟化技术将一台计算机虚拟为多台逻辑计算机。在一台计算机上同时运行多个逻辑计算机，每个逻辑计算机可运行不同的操作系统，并且应用程序都可以在相互独立的空间内运行而互不影响，从而显著提高计算机的工作效率。==> 虚拟化是一个概念

2. 什么是虚拟化技术？
>是一种资源管理技术，是将计算机的各种实体资源，如服务器、网络、内存及存储等，予以抽象、转换后呈现出来，打破实体结构间的不可切割的障碍，使用户可以比原本的组态更好的方式来应用这些资源！
>
>==> 虚拟化技术是虚拟化概念实现的解决方案。典型的虚拟化技术代表有：
> 1. 在硬件抽象层基于 Hypervisors(虚拟机管理程序) 方式来衍生、运行、销毁各种不同操作系统的虚拟机：
>       - Xen，KVM，VMWare，QUEM
>       - Hyper-V(微软提出的一种系统管理程序虚拟化技术)
> 2. 基于 cgroup 和 namespaces 容器(Linux Container，缩写：LXC)

> 通过虚拟化技术模拟的具有完整硬件系统功能的、运行在一个完全隔离环境中的完整计算机系统。==> 虚拟机是虚拟化概念的产物！
