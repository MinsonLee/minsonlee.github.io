## 什么是 Java？

Java 是一种通用、面向对象的编程语言。于 1996 年年初由 Sun Microsystems 公司（于 2009.4.20 被 Oracle 公司以 74 亿美元的价格收购）首次发布 1.0 版本，至今发展为了一种被广泛使用的“工程级”编程语言。

Java 设计者 James Gosling 和其同事 Henry McGilton 于 1995.10 月联合发表了[“Java”白皮书-《The Java Language Environment：A White Paper》](https://www.oracle.com/java/technologies/language-environment.html)（白皮书 PDF [下载](https://www.stroustrup.com/1995_Java_whitepaper.pdf)）其中详细的介绍了 Java 语言的设计初衷、理念等信息，可以借助 [`ChatPDF`](https://www.chatpdf.com/) 快速的阅读一番。

Sun 公司根据针对 Java 的不同使用群体，主要退出了 3 个版本的 Java：

- JSE（Java Standard Edition）：标准版，用于桌面或简单服务器应用开发
- JEE（Java Enterprise Edition）：企业版，基于 JSE 进行构建，提供了更加丰富的用户界面、安全模型、内置网络协议、JSP 等支持，用于复杂服务器应用开发
- JME（Java Micro Edition）：微型版，用于小型设备应用开发，如为移动设备或嵌入式设备开发应用

没有特别说明的情况下，基本我们口头说的 Java8、Java17...其实就是指 `Java SE` 标准版，学习和使用的也是 `JSE` 的版本。

[Java 的官方文档](https://docs.oracle.com/en/java/index.html) 可以找到各种版本的 Java 文档，但是文档结构有些拉胯，也可能是我不会用...

早先时期，开发者根据编程语言与基础硬件的通信方式，将编程语言分为两大类：

1. 编译器：完整的程序通过编译器使用类似英语的自然语法编写，然后，使用该语言将整个代码编译（或翻译）到机器代码中。然后，编译后的代码可以在硬件中运行。
2. 解释器：使用解释器可以实时将每个高级代码语句解释为机器代码。  在查看下一个语句前，硬件可以迅速运行编写的语句。

随着编程的发展，出现了 「字节码」的概念，`Java` 就是这么一门“即时解释型语言”， 即：「源代码-`*.java`」**编译** 为「字节码-`*.class`」，**运行时** 再由「字节码-`*.class`」**翻译** 为「机器代码-二进制代码」。

## JAR、JDK、JRE、JVM 的区别？

简单概述如下：

- `JAR`：`Java`代码文件的集合压缩包
- `JDK` = JRE + Java 自带的工具包 + Java 源码
- `JRE`= JVM + Java API
- `JVM`： `Java` 虚拟机，用于解释运行字节码文件

详细的解释如下：

`JAR`（`Java Archive`-「Java 归档文件」的英文缩写）是与平台无关的文件格式，它允许将许多文件组合成一个压缩文件，为 `J2EE` 应用程序创建的 `JAR` 文件是 `EAR` 文件（企业 `JAR` 文件）。简单来说：`JAR` 就是一份代码集合包，里面包含了项目运行所需的所有自身及依赖扩展包的 `Java` 代码文件。

而在阐述 `JDK/JRE/JVM` 三者区别之前需要简单的说一下：什么是「运行时环境」？

软件程序运行的时候是需要依赖：硬件资源、内存资源、类库、工具包等软件依赖资源...应用程序运行时其本身与这些其依赖资源构成了一个完整的生态，我们将这一生态统称为 “软件运行时环境”，该环境会解决提供对内存和其他系统资源（例如程序文件和依赖项）的访问。

以前，大多数软件直接使用操作系统作为其运行时环境（即：操作系统本身会提供 API 解决软件与资源的交互），但这意味着以前的开发人员必须为他们希望应用程序在其中运行的每个操作系统编写不同的代码，为了解决这一问题 Java 提出了 JRE 这一技术。

`JDK/JRE/JVM` 是成功开发-构建-运行一个 `Java` 程序所必须的三种 `Java` 平台组件：

- JDK（Java Development ToolKit）：Java 开发工具包，即：简单版本的 Java SDK。通俗的来说就是：提供给开发者快速使用并构建出 Java 应用的一整套生态体系（开发平台工具集合，包括了编译器、标准类库、调试工具、堆栈跟踪程序...等工具集）。
- JRE（Java Runtime Enviroment）：Java 运行时环境，即：其包含了 Java 应用程序运行所必需的工具、类库、组件（Java 虚拟机 + Java SE 组件类库）
- JVM（Java Virtual Machine）：Java 虚拟机，充当 `Java` 平台和基础机器硬件之间的额外抽象层，负责对字节码文件进行解释执行。

**由于安装不同操作系统的 JDK 环境，其中包含的 JVM 可以在不同的操作系统上运行，从而保证了 Java 这门编程语言的跨平台性。**

以下简单的列举了几个常用需要了解的工具包功能：

- javac ： 编译器，将 `*.java` 的源代码文件编译为 `*.class` 的字节码文件
- java ： 运行工具，运行 `*.class` 字节码文件，将其解释为机器代码
- javadoc ： 文档生成器，从源码中提取符合规范的注释，生成文档
- jar ：打包工具，将相关的类文件打包成一个文件

## Oracle JDK 、 OpenJDK、GraalVM 的区别？

`JDK` 指的是 Java 开放工具包，在说可以将其理解为是一个规范的概念。由于 Java 的源代码开源，因此出现了很多版本的 `JDK` 安装包，如：`Zulu OpenJDK/OpenJDK/Oracle JDK/GraalVM CE` 这些底层一样，但又各有千秋的 `JDK` 发行包。

就像 `Linux` 内核代码开源后，出现了 `RedHat/Debian/Fedora/Ubuntu` 各类发行版一样。


> 知乎大神 RednaxelaFX  关于「OracleJDK 和 OpenJDK 有什么区别？」 的详细回答：https://segmentfault.com/a/1190000040735603

- 两者都是官方（以前是 Sun 公司，现在是 Oracle 公司）开发维护提供的 Java SE SDK 实现
- 2006 年 Sun 公司在 JavaOne 大会上称将会对 Java 开放源码，2009.4.25 日便发布了 [OpenJDK](https://openjdk.org/) 这一 JDK 的开源版本实现
- 两者遵守的开源协议不一样，OpenJDK 允许开发人员使用、修改、发布版本；而 一般说的 JDK 指的都是 [Oracle JDK](https://www.oracle.com/java/technologies/downloads/)，它只允许开发人员使用，不允许未经许可进行修改和发布
- Oracle JDK 中含有部分 Oracle 公司提供的商业功能和工具，而 OpenJDK 不含有
- 但不管是 OpenJDK 还是 Oracle JDK，其底层的 `JVM` 都是使用 `Hotspot VM`，因此单从运行的角度来说两者没有区别

后面看官方文档的时候，又发现一个 **`GraalVM for JDK`** 的东西：

简单来说这是一个：由 Oracle 公司开发，旨在建立一个能消除编程语言隔离，打通不同语言之间调用的鸿沟，不同语言可以相互共享运行时的 **通用虚拟机**。

- 实现在同一个虚拟机中编译多种语言，如：Java/Ruby/R/JavaScript/Python/Kotlin/Scala/C++
- 实现在同一个语言脚本中调用其他语言的类库，如：在 Java 程序中调用 JS 的函数
- 实现将语言混合调用的代码编译为 1 个高效、运行占用资源少的二进制可执行程序

## JDK 和 Java EE SDK 的区别？

JDK 本身是不区分 `SE/EE/ME` 版本的，因此只需要安装 [Java SE 对应版本的 JDK](https://www.oracle.com/java/technologies/downloads/) 即可。但是在看官方文档时还发现了 [`Java EE SDK`](https://www.oracle.com/java/technologies/java-ee-sdk-download.html) 的东西，有点好奇两者有什么不一样的吗？

翻看[官方文档的 `Installing the Software` 部分](https://www.oracle.com/java/technologies/javaee7sdk-install.html) 后大致的明白了，`Java EE SDK` 是一个依赖于 `JDK` 的 `SDK` 包，用于补充 `JDK` 的功能，使得可以直接构建和开发 `JEE` 的 `Java` 应用。

## 国内标题党产生的误解：Java 收费问题小解

结论：`Oracle` 公司提供了免费的 `Oracle JDK` 供开发者**使用**，同时提供了免费的 `OpenJDK` 公开发者使用和修改发布。

起因：`2019` 年发布 `Java 11` 发布，`Oracle` 公司也顺带修改了 `Oracle JDK 8` 的用户使用协议，大意是：

1. **如果在某个版本的 `Oracle JDK` 在其开放维护期结束后出现了 Bug 或 安全漏洞，用户可以通过「付费」的方式从 `Oracle` 公司得到技术支持，对这个已经停止公开维护了的 `JDK` 进行更新维护**
2. **即使你从其它非官方处得到了某一个已经停止了公开维护的 `JDK` 官方更新，但如果企业想要使用这一更新并用于商业之上，需要付费购买商业许可证**

就这样被国内很多标题党捕风捉影的传递为了 “Java 要收费了”...

那么如果你使用已经停止公开维护的 `Oracle JDK` 真的出现了问题，你其实可以选择：

1. 升级到当前免费公开维护的 `Oracle JDK` 版本中。因为每一个新版本都会对上一个版本进行 Bug及漏洞修复、特性扩展、语言优化等操作，但是 `JDK` 现在变成了每半年一版本更新。对于用 `Oracle JDK` 的企业来说就意味着其每半年需要升级一次 `Oracle JDK` 的版本...这对于企业来说是不想做的，因为每一次升级都意味着“风险” 。
2. 考虑将 `Oracle JDK` 换成 `OpenJDK`。如果你只是单纯的使用，并没有用到 `Oracle JDK` 的一些特有类库之类的，那么完全可以用 `OpenJDK` 进行平替。

### 2. 手动安装 OpenJDK

```sh
# 下载 openjdk 源码包：其他版本可去 https://openjdk.org/ 找

# 解压源码

mv /apps/jdk-17 /apps/openjdk-17

# 设置 Java_HOME 环境变量（vi /etc/profile，加入下列 3 行）
export JDK_17=/apps/openjdk-17
export JAVA_HOME=${JDK_17}
export PATH=${JAVA_HOME}/bin:$PATH

# 在JDK 1.5 之后，完全可以不配置 JRE_HOME/CLASSPATH 变量
# 不配置 CLASSPATH，JRE 会自动搜索当前路径下的类文件
# 编译、运行时，系统可以自动加载 dt.jar 和 tools.jar 文件中的 Java 类。
# 如果遇见教程让配置classpath的，那么大概率教程陈旧，不看也罢
# export JRE_HOME=${JAVA_HOME}/jre
# export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib


# 重载配置文件，立即生效
source /etc/profile

# 测试 JDK 是否安装成功
java --version
javac --version
```

## Java 的基本程序设计结构

以下是一个 Demo 程序：打印 `JAVA_HOME` 环境变量的值：

```java
public class Main {
    public static void main(String[] args) {
        String javaHome = System.getenv("JAVA_HOME");
        System.out.println("JAVA_HOME: " + javaHome);
    }
}
```

- `Java` 是严格面向对象的编程语言，所有的程序都需要以类的方式进行编写
    - 类定义方式：访问限制符 class 类名 {}
    - 类名必须是字母开头接任意长度的字母和数字，推荐以驼峰命名法进行命名
    - 主入口函数：一定是 `main` 函数，且入口 `main` 函数固定格式是 `public static void main(String[] args) { /*结构体*/ }`
- `Java` 严格区分大小写
- 访问修饰符（access modifier）用于控制程序的其他部分对该段代码的访问权限
    - public 公有，所有都可以访问
    - protected 受保护的，子类可以单向访问存在继承关系的父类中的 protected 属性和方法
    - private 私有的，有且只有在自己内部可以进行访问


--- | Java | PHPP
---|---|---
标识符 | `[a-zA-Z0-9_$]` | `[a-zA-Z_\x80-\xff]`
变量命名 | `^[a-zA-Z_$][a-zA-Z0-9_$]+`| `^$[a-zA-Z_][a-zA-Z0-9_]*$`
编译类型 | 半解释性、强类型、静态语言 | 解释性、弱类型、动态脚本语言
编程范式 | 面向对象 | 面向对象、面向过程、函数式编程、[元编程](https://houbb.github.io/2023/08/29/awesome-java-cn)


相同点：
- 大小写敏感
- 关键字不能作为命名
- 注释支持单行、多行
- 面向对象编程的基本特性，注意：Java 支持方法重载，PHP 没有方法重载
- 访问控制：private、protected（Go 没有）、public

Java 源文件声明规则

- 一个源文件只能有且只有 1 个 public 类，可以有多个非 public 类
- 源文件的名称应该和 public 类的类名保持一致
- 程序都是从main方法开始执行。

## Java 中一些分层的概念

### EJB、POJO、JavaBeans、ORM、DAO、DO、BO、DTO、VO、AO、PO、UID 名词意义

![领域模型中的实体类](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/entity-classes-in-domain-model.png)

- `Java Beans`
- `EJB（Enterprise Java Beans）`：企业 Java Beans 的英文缩写。
- `POJO（Plain Ordinary Java Object）`：简单的普通 Java 对象的英文缩写。
- `ORM（Object Relational Mapping）`：对象关系模型。是一种将数据库实体映射成对象的概念，可以理解为：一个对象就是数据库中的一个表，通过访问对象的方式来访问数据表的信息
- `DO（Data Object）`：与数据库表结构一一对应，通过 DAO 层向上传输资源对象
- `DAO(Data access object)`： 数据访问对象。通过统一的 API 对象来操作不同类型的数据库，使得上层代码不需要考虑数据引擎或表的关系
- `DTO（ Data Transfer Object）`：数据传输对象，Service 或 Manager 向外传输的对象
- `BO(Business Object) `：业务对象，由 Service 层输出的封装业务逻辑的对象，把业务逻辑封装为一个对象（这是一个复杂对象，里面可能包含多个类）
- `AO（ Application Object）`：应用对象。 在Web层与Service层之间抽象的复用对象模型，极为贴近展示层，复用度不高
- `VO（ View Object）`：显示层对象，通常是Web向模板渲染引擎层传输的对象
