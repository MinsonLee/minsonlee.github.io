面向对象编程（Object-Oriented Programming, OOP）和函数式编程（Functional Programming, FP）是两种不同的编程范式。

- 面向对象：着重于将现实世界的问题域分解并进行抽象为相互关联的对象（数据+操作），通过封装、继承和多态等机制来组织和管理这些对象。**注重：谁+做什么+怎么做，“谁”会因为操作而发生改变**
- 函数编程：强调程序是一系列数学函数的组合，这些函数仅依赖于其输入并产生确定的输出，不改变外部状态（无副作用）。**注重：“做什么”而非“怎么做”**

`Java` 是一个纯面向对象的语言，但在现实中很多注重“行为”的场景（函数式编程），如果我们依然通过 `OOP` 的方式反而不好进行抽象。

在 `PHP` 中我们可以通过将一个闭包函数赋值给一个变量，然后通过进行调用，进而来解决这种“注重行为”的场景。
在 `Java` 中为了解决这一问题，

`@FunctionalInterface` 注解是 `Java 8` 引入的一个注解，专门用于标记一个接口为函数式接口。


- 常用的函数式接口：`java.util.function.*`
    - `java.util.function.Consumer<T>` : 定义了一个接受 `<T>` 类型参数没有返回值的抽象方法
    - `java.util.function.Predicate<T>` : 定义了一个接受 `<T>` 类型参数，固定返回布尔值的抽象方法
    - `java.util.function.Function<T, R>` : 定义了一个接受 `<T>` 类型参数并返回 `<R>` 类型结果的抽象方法。通过 `R apply(T t)` 调用
    - `java.util.function.BiFunction<T, U, R>` : 定义了一个可以接收两个参数 `<T>` 和 `<U>` 类型并返回 `<R>` 类型结果的抽象方法。通过 `R apply(T t, U u);` 调用


```java
// 校验方法
BiFunction<InsuranceDisableConfigVo.Disable, String, Boolean> check = (config, key) -> {
    // 如果没有配置的纬度
    if (
        config == null
        || !DISABLE_CONFIG_FIELDS.contains(key)
        || !paramMap.containsKey(key)
        || ReflectUtil.getFieldValue(config, key) == null
    ) {
        return true;
    }
    // 没有设置该屏蔽规则、屏蔽规则是 0 或 空字符串 ==> 无限制
    // 屏蔽规则是数组 ==> 数组内的值，则屏蔽
    // 屏蔽规则非数组，则判断相等则
    Object configValues = ReflectUtil.getFieldValue(config, key);
    if (configValues instanceof List) {
        List<String> configValuesList = (List<String>) configValues;
        return CollectionUtils.isEmpty(configValuesList) || paramMap.get(key) != null && configValuesList.contains(paramMap.get(key));
    } else if (configValues instanceof String) {
        String configValuesString = (String) configValues;
        return "0".equals(configValuesString)
            || "".equals(configValuesString)
            || (paramMap.get(key) != null && configValuesString.equals(paramMap.get(key)));
    }

    return false;
};

if (CollectionUtils.isNotEmpty(disableConfigs)) {
    for (InsuranceDisableConfigVo.Disable item : disableConfigs) {
        if (
            check.apply(item, "channel") &&
            check.apply(item, "dealerId") &&
            check.apply(item, "citizenCountryCode") &&
            check.apply(item, "insuranceType") &&
            check.apply(item, "pickupRegion") &&
            check.apply(item, "pickupCity") &&
            check.apply(item, "pickupStoreId") &&
            check.apply(item, "site")
        ) {
            return true;
        }
    }
}
```
