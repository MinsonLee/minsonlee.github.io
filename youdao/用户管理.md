
## 用户

参考：[Linux用户和用户组管理详解](http://c.biancheng.net/linux_tutorial/60/)

- `/etc/default/useradd` 系统添加用户时候（`useradd`）时的默认信息配置，可以使用 `useradd -D` 查看
- `/etc/passwd` 该文件用于用户登录时校验 用户的口令,当然应当仅对root可写.
- `/etc/shadow` 存储 Linux 系统中用户的密码信息（只有 root 用户拥有读权限）
- `/ect/group` 文件是用户组配置文件


> PS：通过 root 用户登录服务，然后执行 su www 切换了用户身份。root 是当前的登录用户，而 www 是当前的有效用户。
> - 登录用户：通过 ssh 命令链接到服务器时使用的用户
> - 有效用户：当前执行命令时所处于的用户身份

- who 打印当前会话的**登录用户**
- who am i 打印当前会话的**登录用户**（相当于 who -m）
- whoami 打印当前**有效用户**的用户名（等同于 echo ${USER}）
- id 打印当前**有效用户**详细用户信息
- useradd -r -g mysql -s /bin/false mysql # 创建 `mysql` 用户：隶属mysql组，禁止 `shell` 登录
- usermod -aG docker ${USER} # 将当前用户加入 docker 用户组


1. 创建用户

```sh
# 语法：useradd [-mMnr][-c <备注>][-d <登入目录>][-e <有效期限>][-f <缓冲天数>][-g <群组>][-G <群组>][-s <登录默认shell>][-u <uid>][用户账号名称]
# -m 自动创建用户登录目录
# -M 不要自动建立用户的登录目录
# -n 取消简历已用户名称为名的群组
# -r 建立系统账号

# 创建用户同时新建用户$HOME目录
# 
# -d </home/path> 指定 HOME 目录
# -u <uid> 指定创建用户的 UID
# -g <group_name> 将创建的用户加入到指定的组
useradd <-u uid> -d /home/lms lms

# 为用户设置密码
passwd lms
```

2. 设置用户ssh登录

- 创建用户时：通过 `-s` 参数进行指定
- 已存在用户：通过 `chsh` 命令来修改，`chsh -s <shell> [username]`

3. 设置用户sudoers

- 1、创建：/etc/sudoers.d/<username>，然后添加一行`<username> ALL=(ALL) NOPASSWD:ALL`
- 2、在 /etc/sudoers 下添加一行 `<username> ALL=(ALL) NOPASSWD:ALL`
- 3、在 /etc/sudoers 下添加一行 `%<userGroup> ALL=(ALL) NOPASSWD:ALL`，如：`%sudoers ALL=(ALL) NOPASSWD:ALL` 将 `sudoers` 组的所用用户都可以 `sudo -i` 切到 `root`
    - https://utho.com/docs/tutorial/how-to-add-user-to-sudoers-or-sudo-group-in-centos-7-support-internal/

**3. uid 变更会导致什么问题？**

使用 NFS 服务在各个机器之间同步静态资源信息（`/data/img`），该目录的宿主用户是 `www`，由于测试环境的 Web 机被挖矿因此重装了系统，导致出现了 web机和其它机器之间的 `www` 用户其 uid 不一致（原：测试环境 `www` 的 `uid` 统一是 `500`，生产是 `504`）。

Linux 的用户最终在 `/etc/passwd` 中其实是映射为一个个 UID 的，而 Linux 目录的权限看起来是 www，但实际目录的权限是跟 UID 进行关联的，导致了 Web 机虽然是 www 用户但是却无法写入 `/data/img` 目录，而将 web 机上的 `/data/img` 目录改为 web 机上的 www 用户后又会导致其他机器在同步 `/data/img` 目录时内容无法写入。
