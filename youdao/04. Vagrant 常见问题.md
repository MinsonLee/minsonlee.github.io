## vagrant up 启动问题

1. **`vagrant up` 的时候卡住报错：`Warning: Authentication failure. Retrying...`**

> 注意，此时虚拟机其实是启动完成了的，只是登陆不上去。你可以打开 VirtualBox 看到是有虚拟机跑起来的

配置 `SSH Username/Password` 不走 `ssh key` 验证

在 `Vagrantfile` 中添加上用户明文登陆信息即可：

```shell
config.ssh.username = "vagrant"
config.ssh.password = "vagrant"
```

然后保存 Vagrantfile，重启 Vagrant：

```shell
$ vagrant up
```

`vagrant ssh` 进入虚拟机，然后更新一下 `.ssh/authorized_keys`，

```sh
wget https://raw.githubusercontent.com/mitchellh/vagrant/master/keys/vagrant.pub -O .ssh/authorized_keys
```

确保文件权限正确

```sh
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
chown -R vagrant:vagrant .ssh
```

重启虚拟机即可！
再重启

7. 启动失败，提示已有同名虚拟机：`A VirtualBox machine with the name 'xxx' already exists.`

打开 `Virtualbox` 的界面看一下一般都会是真的已经存在了一个同名虚拟机，做法有两种：

- 方法 1： 手动删除已经存在的虚拟机
- 方法 2： 通过 `vboxmanage list vms`，然后将对应的 `id` 拷贝替换 `./.vagrant/machines/default/virtualbox/id` 这个文件中的虚拟机 `ID` 即可。


8. 在启动过程中，提示 `vagrant` 和 `VitrualBox` 版本不对：`The guest additions on this VM do not match the installed version of VirtualBox! In most cases this is fine, but in rare cases it can prevent things such as shared folders from woking preperly. If you see shared folder errors, please make sure the guest additions within the virtual machine match the version of VirtualBox you have installed on your host and reload your VM.`

`Guest Additions` 是虚拟机核心系统安装好，额外需要安装的驱动及其优化程序。`vagrant box update` 以后又没有更新 `VirtualBox` 往往可能是这个问题的主要原因。

这个问题有时候可能会导致直接启动系统失败，也可能导致**加载共享文件失败**...但是这个只是一个「提示信息」，所以需要细致的看看才能发现。

> Vagrant was unable to mount VirtualBox shared folders. This is usually because the filesystem "vboxsf" is not avaiable. This filesystem is made avaliable via the VirtualBox Guest Additions and kernel module.

- 方式1：重装最新的 VirtualBox
- 方法2：`vagrant plugin install vagrant-vbguest` 通过插件解决，每次 `vagrant up` 的时候都会自动检查 `Guest Additions`，如果发现过期了，就会自动更新。

## 插件工具

安装完插件记得 `vagrant up/reload` 一下

### `Guest Addtions` 自动更新

`vagrant plugin install vagrant-vbguest` 自动更新 `Guest Addtions` 和内核的匹配信息

### `Hosts` 自动更新

`vagrant plugin install vagrant-hostupdater` 可以自动处理 `yaml` 中的 `ip/hosts` 映射更新到宿主机的 `hosts` 文件中（这样就可以不用每次更新了 `yaml` 的站点配置后还需要手动更改宿主机的 `hosts`）

> 喜欢手动处理的推荐通过 `SwitchHosts` 工具来管理

该工具会在 **vagrant up(启动)/resume(唤醒)/reload(重启)** 的时候自动添加 `hosts`，每次 **halt(关机)/destroy(销毁)/suspend(睡眠)** 的时候自动删除掉它给你添加的 `hosts`

### Homestead Box 实现内网传统，分享你的站点

`vagrant share [--http 8000]` 命令可以为本地开发中的站点，通过在 `vagrant cloud` 上创建供外网访问的 URL。

如果第一次执行会提示你通过 `vagrant login` 登录你的 [`vagrant cloud account`](https://atlas.hashipcorp.com/account/new)，如果已经登录过那么它会在你的本地保存一个 `access token` 用来保存和验证你的登录状态，从而 避免你每次执行 `vagrant share` 的是都需要输入用户/密码。

- 默认共享转发虚拟机中的 `80` 端口，可以通过 `--http <port>` 来指定它转发的端口
- 默认随机分配二级域名（即：xxxx.vagrantshare.com），可以通过 `--name=<domain>` 指定二级域名`xxx`的部分

`vagrant share` 只适合短暂分享单站点，如果你的虚拟机中部署了多个站点，那么你需要注释掉 `yaml` 文件中的其他站点，然后 `vagrant reload --provision` 来重新编译再 `vagrant share` 你的站点。

如果你部署了多个站点，你又不想重新编译。那么...如果你的 `Vagrant/Homestead` 版本比较新，那么可以通过 `vagrant ssh` 进入虚拟机，然后进到对应的项目执行 `share xxxx` 就可以。
