## 算术运算符
算术运算符**主要**是对**数值类型变量**进行运算的。如：加减乘除

| 运算符 |         备注         | 范例 | 结果 |
|--------|----------------------|------|------|
|   `+`  |正号（表示某数为正数）| +3   | +3   |
|   `-`  |负号（表示某数为负数）| -3   | -3   |
|   `+`  |加号（表示进行加操作）| 5+5  | 10   |
|   `-`  |减号（表示进行减操作）| 5-5  | 0    |
|   `*`  |乘号（表示进行相乘操作）| 5\*5 | 25 |
|   `\`  |除号（表示进行相除操作）| 5\*5 | 25 |
|   `%`  | 取模(取余数)         | 7%5  |  2   |
|  `++`  | 自增(i=i+1)          | a=2;a++ | 3 |
|  `--`  | 自增(i=i-1)          | a=2;a-- | 1 |
|  `+`   | 字符串拼接符         | "He"+"llo" | "Hello" |

### 算数运算符使用注意细节
1. 对于除号`/`：①**若除数和被除数都是整数，得到的商只保留整数部分，会舍弃掉小数部分**；② ==**若希望商保留小数部分，则需要有浮点数参与运算**==
    ```golang
    var n1 float32 = 10/4
    fmt.Println(n1) // 2
    
    var n2 float32 = 10/4.0
    fmt.Println(n2) // 2.5
    ```
    

2. **Golang 中的取模运算是取余操作**,公式：`a-((a/b)*b)`
    - 在 `C/C++`、`C#`、`JAVA`、`PHP`这些主流语言中`%`运算符都是做**取余运算**，而在`Python`中`%`是做**取模运算**
    ```golang
    fmt.Println( 5 % -3) // 输出：2；等价于：-((5/-3)*-3)
    ```
    ```python
    # -*- coding: UTF-8 -*-
    // Python 中的取模运算
    print( 5 % -3) // 输出 -1
    ```


3. Golang 中自增`++`/自减`--`**只能当做一个独立的语句使用**
    - `i++`等价于`i=i+1`/`i--`等价于`i=i-1`，因此不能将自增/自减运算符进行赋值操作：`a = i++` 或 `a := i--`
    - 只能当做独立的语句进行使用
    ```golang
    i := 1
    // 以下两段代码编译都会报错，不通过
    if i++ > 0 {
        fmt.Println("This is error")
    }
    
    fmt.Println( i++ )
    ```


4. Golang 中 `++` `--` 都**只能写在变量的后面**。即：==只有`a++`和`a--`，没有`++a`或`--a`==
> 由于 `++` 和 `--` 只能作为**独立语句使用**，那么`a++`(先运行当前语句再执行自增操作)和`++a`(先自增操作再运行当前语句)两者便没有区别了


5. Golang 在设计时去掉了 `C/Java` 中自增自减容易混淆的写法，让 Golang 的语法更加简洁、统一
> **强制性：一个问题尽量只有一种解决办法**




## 关系运算符(比较运算符)
- 关系运算符的结果都是 `bool` 型,也就是：要么`true`,要么`flase`
- 关系表达式常用于：**条件判断语句**和**循环语句**中

| 运算符 |   备注   |      范例      |  结果  |
|--------|----------|----------------|--------|
|  `==`  | 相等于   | 4 == 3         | flase  |
|  `!=`  | 不等于   | 4 != 3         | true   |
|  `>`   | 大于     | 4 > 3          | true   |
|  `<`   | 小于     | 4 < 3          | false  |
|  `>=`  | 大于等于 | 4 >= 3         | true   |
|  `<=`  | 小于等于 | 4 <= 3         | false  |

### 关系运算符使用注意细节
1. **关系运算符的结果都是 `bool` 型**,也就是：要么`true`,要么`flase`
2. 关系运算符组成的表达式，称之为：关系表达式。如：`a > b`
3. ！！！注意：①相等于`==`不要错写为赋值运算符`=`；②`4 > 4`的结果是`false`




## 逻辑运算符
逻辑运算符**只能用于连接多个条件表达式**，最终结果也是`bool`型

假定：A 值为 true, B 值为 false
| 运算符 | 描述 | 实例 |
|--------|------|------|
| `&&` | 逻辑与：有且仅有运算符两边的值都为`true`，逻辑为`true`,其余情况为 `false` | `(A && B)` 为 `false` |
| `||` | 逻辑或：有且仅有运算符两边的值都为`false`,逻辑为`false`,其余情况为 `true` | `(A || B)` 为 `true` |
| `!` | 逻辑非：若条件为`true`,则逻辑为`false` | `!(A && B)` 为 `true` |

### 逻辑运算符使用注意细节
1. 在 Golang 中逻辑运算符**只能用于连接多个条件表达式**，最终结果也是`bool`型
```golang
// 编译报错：invalid operation: i == 1 && a (mismatched types bool and int)
i, a := 1, 1
if (i==1) && a {
    fmt.Println("编译不通过，会报错!")
}
```

2. 逻辑与`&&`、逻辑或`||`，又==称之为`短路运算`==
```golang
func main() {
    var i int = 10
	// 定义闭包匿名函数
	test := func () bool {
		fmt.Println("Here is a test print")
		return true
	}
	// 短路与：有且仅有第一个条件为 true，第二个条件才会进行判断
	if i < 9 && test() {
		fmt.Println("OK") // 不会输出任何东西
	}
	// 短路或：有且仅有第一个条件为false时，第二个条件才会进行判断
	if i < 9 || test() {
		fmt.Println("hello") // 先输出`Here is a test print!`,再输出`hello`
	}
	if i > 9 || test() {
		fmt.Println("hello") // 只输出`hello`
	}
}
```

3. Golang 中的短路运算两边只能连接 bool 类型的数据类型,如下：在PHP中是可行的，但在 Golang 中编译是会报错的
```php
<?php
$a = 1;
if ($a == 1 && $a++)
{
    var_dump($a); // 打印：int(2)
}
```

```golang
a := 1
if a == 1 && a++ { // Golang 中直接编译不通过报错
    fmt.Println(a)
}
```


## 赋值运算符
赋值运算符的作用是将某个运算的值赋给某个指定的**左值**

| 运算符 |   备注   |      范例      |
|--------|----------|----------------|
|  `=`  | 简单的赋值运算符，将一个表达式的值赋给左值 |` C = A + B` 将 `A+B`运算的结果赋给 `C`|
|  `+=`  | 相加后再赋值| `C += A` 等价于 `C = C + A`| 
|  `-=`  | 相减后再赋值| `C -= A` 等价于 `C = C - A`| 
|  `*=`  | 相乘后再赋值| `C *= A` 等价于 `C = C * A`| 
|  `/=`  | 相除后再赋值| `C /= A` 等价于 `C = C / A`| 
|  `%=`  | 取模后再赋值| `C %= A` 等价于 `C = C % A`| 


| 运算符 |   备注   |      范例      |
|--------|----------|----------------|
| `<<=`  | 左移后再赋值| `C <<= 2` 等价于 `C = C << 2`| 
| `<<=`  | 右移后再赋值| `C >>= 2` 等价于 `C = C >> 2`| 
|  `&=`  | 按位与后再赋值| `C &= 2` 等价于 `C = C & 2`| 
|  `^=`  | 按位异或后再赋值| `C ^= 2` 等价于 `C = C ^ 2`| 
|  `|=`  | 按位或后再赋值| `C |= 2` 等价于 `C = C | 2`|

### 赋值运算符使用案例与注意事项
案例：将 a b 两个变量的值进行交换，并不依靠其他变量
```golang
package main
import "fmt"
func main() {
    var a int = 1
    var b int = 2
    // 方法1：golang 语法本身的支持
    a, b = b, a
    fmt.Printf("a=%v b=%v", a,b)
    
    // 利于赋值运算符
    a += b
	b = a - b
	a -= b
	fmt.Printf("a=%v b=%v", a,b)
}
```

1. 赋值运算符的执行顺序是：**从右至左**
2. 赋值运算符的左边，只能是**变量**，右边可以是：**变量**、**表达式**、**常量(字面量)**
3. 复合赋值运算都可以拆解为对应的等价简单运算


## 位运算符
> - 位运算与二进制相关
> - 位运算符都是双目运算符

| 运算符 |   描述   |   运算规则     |
|--------|----------|----------------|
|  `&`  | 按位与运算符| 有且仅有相同二进位都为1为1，否则为0| 
|  `|`  | 按位或运算符 | 相同二进位有一个为1则为1，否则为0 | 
|  `^`  | 按位异或运算符 | 相同二进位不同则为1，否则为0| 
| `<<`  | 左移运算符 | 把左边的运算数的各个二进位全部左移若干位，高位丢弃，低位补0【左移n位就是乘以2的n次方】| 
| `<<`  | 右移运算符 | 把左边的运算数的各个二进位全部右移若干位，高位补0，低位丢弃【右移n位就是除以2的n次方】|

## 运算符的优先级
- 所谓优先级，即：表达式运算的先后顺序，==运算符优先级高的表达式先运算==。如下表：**上一行的运算优先级总比下一行的要高（优先级由高至低）**
- **只有单目运算符、赋值运算符是从右到左运算**

| 分类 | 描述 | 关联性(运算顺序) |
|------|------|--------|
| 后缀 | `()` `[]` `->` `.` `++` `--` | 左到右  |
| 单目 | `+` `-` `!` `(type)` `*` `&` `sizeof()` | ==右到左== |
| 乘法 | `*` `/` `%` | 左到右 |
| 加法 | `+` `-` | 左到右 |
| 移位 | `<<` `>>` | 左到右 |
| 关系 | `<` `>` `<=` `>=` | 左到右 |
| 相等(关系) | `==` `!=` | 左到右 |
| 位与 | `&` | 左到右 |
| 位异或 | `^` | 左到右 |
| 位或 | `|` | 左到右 |
| 逻辑与 | `&&` | 左到右 |
| 逻辑或 | `||` | 左到右 |
| 赋值 | `=` `+=` `-=` `*=` `/=` `%=` `>>=` `<<=` `&=` `^=` `|=` | ==右到左==|
| 逗号 | `,` | 左到右 |

> 有复杂运算时，最好使用括号括起来，这样就不会担心由于优先级导致结果不是期望的值
