# 替换 python 命令为 python3
[TOC]
> 当前(2019/11/22)，Linux Ubuntu 18.04 中 python 的默认版本任然是 2.x 版本，Python3.x 版本的命令为 python3，个人常用学习的也已经是 3.x 版本.
> 因此想将 python 命令默认为启动 python3，特此记录设置方法。修改其他命令同理

- [x] 2023/08/29 补充：不要改动系统的 Python 默认版本，否则可能会出现一些系统依赖组件无法使用（如 CentOS 的 yum 源）

## alias bash 方式
> 如果只是针对当前用户、当次进程内的操作进行更改（即：重新登录后恢复原状），可以使用这种方法。

```sh
$ alias python=python3
```

## 通过 .bashrc 文件
> 如果希望该更改是针对`当前用户永久生效`，可以通过修改 `~/.bashrc` 方式操作

1. 编辑 ~/.bashrc 文件，将 `alias python=python3` 添加到配置中
```sh
# 直接追加到 ~/.bashrc 文件底部
echo alias python=python3 >> ~/.bashrc
```
```sh
# vim 编辑添加，添加结果如下图
vi ~/.bashrc
```
![image](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/0B5FBBBEAC744097BC23553EB417B8FC.png)

> 从 ~/.bashrc 文件的源码中可看到，若 ~/.bash_aliases 文件存在，会被自动加载进来！因此我们也可以将自定义的 alias 别名命令添加到  ~/.bash_aliases 文件中

2. 重新初始化配置文件
```sh
$ source ~/.bashrc
```
>  source 命令也称为“点命令”，也就是一个点符号（.）,是 bash 的内部命令。功能是：重新执行指定的 bash 初始化文件，使之在当前进程中立即生效，而不必注销并重新登录。

## 修改 python 命令软连
> 如果你想从针对所有用户修改掉，可以通过修改 python 命令的链接文件。

1. 查看当前 python 命令版本
```sh
python --version
```
> 若出现 `Command 'python' not found, did you mean` 则意味着该命令没有被创建软链文件

2. 查看当前已安装的 python 版本和软链接情况
```sh
$ ll /usr/bin/python*
```

![image](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/yd-753086154B114B73BF1FFDEDD00A2BD8.png)

3. 若存在 python 的软链，删除原有的 python 软链文件
```sh
# 为了防止后续需要使用旧版本 python，可以备份一下软连为 python2
$ ln -s /usr/bin/python /usr/bin/python2

# 删除软链文件
$ sudo rm /usr/bin/python
```

4. 建立指向 Python3.X 的软链
```sh
$ ln -s /usr/bin/python3 /usr/bin/python
```

> 如果执行未成功，可以看一下系统环境变量是否添加 `/usr/bin/` 目录，查看系统环境变量
```sh
echo $PATH
```

![echo $PATH](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/echo-PATH.png)

或
```sh
# export 命令
$ export
```
![linux print export command](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/linux-export-command.png)

> 把路径 `/usr/bin` 加入环境变量 PATH 中（一般来说已经包含了该目录），若无 vim 修改 /etc/profile 添加即可，如下图

![export PATH](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/export-path.png)

## 如何在不修改系统 Python 版本的情况下，特定情况使用 Python3

1. **系统保持Python2**

    不要更改系统级别的Python设置，让其默认为Python2。

2. **为`cephadm`使用Python3**

    - **方法1: 创建符号链接**
        
        在`cephadm`所在目录创建一个Python3的符号链接。

        ```bash
        ln -s /usr/bin/python3 /path/to/cephadm-directory/python
        ```

    - **方法2: 修改脚本头部**
        
        打开`cephadm`脚本，将第一行（Shebang）改为Python3的路径。

        ```bash
        #!/usr/bin/python3
        ```
    
    - **方法3: 环境变量**
        
        在运行`cephadm`之前，设置`PYTHON`环境变量指向Python3。

        ```bash
        export PYTHON=/usr/bin/python3
        ```
    
3. **测试**

    运行`cephadm`以确认它正在使用Python3。

这样，你的系统会默认使用Python2，而`cephadm`会使用Python3。注意测试所有更改以确保没有破坏任何功能。
