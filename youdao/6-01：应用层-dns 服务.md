## 域名

### 概念

#### 什么是域名？
`域名`是 Internet 网络上的一个服务器或一个网络系统的名称，形式是以若干个字母`或`数字组成，以 "." (英文状态的点) 进行分隔成几部分。

#### 什么是 FQUN ？
FQDN 是 Fully Qualified Domain Name 的英文首字母缩写，中文全称为：完全限定域名。
一个完全限定域名会同时带有`主机名`和`域名`的名称，例如：主机名是 baike,域名是 .baidu.com. ,那么FQDN就是 baike.baidu.com.

#### 域名的组成？
> PS：以 baike.baidu.com. (百度百科)域名为例
- 根域名：. 【早期的域名必须以英文句号“.”结尾，例如，当用户访问 www.wikipedia.org 的 HTTP 服务时必须在地址栏中输入：http://www.wikipedia.org. ，这样DNS才能够进行域名解析。如今DNS服务器已经可以自动补上结尾的句号，因此一般情况我们忽略不写】
- 顶级域名（）：.com
- 二级域名：.badidu ==> 全球唯一
- 多级域名(三级/四级...)：baike

## DNS（域名系统）

### 概念

#### 什么是 DNS ？
- DNS 是 Domain Name System 的缩写，中文名称是：域名系统，是互联网的一项服务。
- 域名系统(DNS) 允许将 DNS 命名空间分成多个区域，每个区域存储有关`一个或多个`的连续 DNS 域的信息，当前，对于每一级域名长度的限制是 63 个字符，域名总长度则不能超过 253 个字符。
- 一般情况下，DNS 使用 `UDP` + `53 端口`【理论上说可以使用 TCP，但 DNS 服务器查询域名，一般返回的内容都不超过 512 字节，用 UDP 查询即可，事实很多 DNS 服务器在进行配置时，也仅支持 UDP 查询包】

#### 什么是区域名称？
- 区域名称指定 `DNS 命名空间`的`部分`，该部分由此服务器管理。这可能是你组织单位的域名（例如，.microsoft.com）或此域名的一部分（例如，.newzone.microsoft.com），甚至你需要还可以将其设置为 .com 。
- 区域名称不是 DNS 服务器的名称。

#### DNS 的作用？
域名系统将`域名`和 `IP地址`相互映射(哈希表)的一个分布式数据库，能够使人更方便地访问互联网(快速的获取域名对应的 IP 地址) ==> 域名解析


#### 什么是泛域名解析？
泛域名解析是指将 `*.域名` 解析到`同一 IP`上，该域名的所有子域名，均会被解析指向到同一个 Web 地址。（例如对域名 a.com 进行泛域名解析设置，也就是说域名 a.com 之下所设的 `*.a.com` 全部解析到同一个 IP 地址上去，比如设 **.a.com 就会自己自动解析到与 a.com 同一个IP地址上去）

#### 什么是根域名服务器
根域名服务器主要用来管理互联网的主目录(.)，截至 2019-06-08 全球共有 985 台根域名服务器(其中)被划分为 13 组根域名服务器逻辑节点（被编号为 A-->M ）被 12 个独立的根服务器运营商运营着。
> - 拓展信息-根服务最新信息：https://root-servers.org/
> - 拓展信息-ICANN 对 `全球有 13 个根服务器`这一错误答案的回复：https://www.icann.org/news/blog/there-are-not-13-root-servers

#### Windows 中安装/配置 DNS 信息

##### Windows Server 上安装 DNS 服务

- Windows Server 2008 及以前版本：`控制面板`(Win+R --> control) ==> 添加或删除程序 ==> 添加/删除 Windows 组件 ==> 网络服务(选中点击“详细信息”) ==> 域名系统(DNS)

> 1. 安装过程可能需要 Windows Server 安装盘，用以提供对应的安装信息
> 2. 以上安装 DNS 服务需要在 Windows Server 2008 版及以前的版本中进行操作

- Windows Server 2012 及以后版本：`服务器管理器` ==> `添加角色和功能` ==> 选择安装类型及服务器选择（`基于角色或基于功能的安装`）==> 服务器选择（`从服务器池中选择服务器`）==> 找到 `DNS服务器` 勾选然后`添加功能`
==> 点击`下一步`到`安装`

##### 配置 DNS 服务器

`服务器管理器` ==> 启动`DNS 管理器` ==> 右键`正向查找区域` ==> 新建区域(例如baidu.com.) ==> 右键新建的区域 ==> 新建主机(如：www.baidu.com. 是 baidu.com. 域下的一个具体主机) ==> 配置FQDN 的 IP 即名称信息

> 此时：
> - 只要有计算机来询问 `www.baidu.com.` 的 IP 地址，该 DNS 服务器就会直接将刚才配置的 IP 地址返回。
> - 若有计算机来询问 `baike.baidu.com.` 的 IP 地址，该 DNS 服务器会返回：`Unknow can't find baike.baidu.com: Non-existent domain`，因为在设置 DNS 配置信息的时候，该 DNS 服务器对整个 `baidu.com.` 区域进行了接管，即使没有该记录也不会往根服务器进行查询。
> - 若有计算机来询问 `www.souhu.com.` 的 IP 地址，该 DNS 服务器发现并没有该记录，则会迭代/递归向根服务器查询该域名 IP 地址信息，查到将对应信息返回并缓存一份(`查看>高级`可以看到该 DNS 服务器缓存的记录)
