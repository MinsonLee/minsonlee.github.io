**算法设计的要求：**

- 正确性：算法能满足并解决具体问题的需求
- 可读性：程序应该是为人们阅读而编写的，只是偶然地供机器执行—— 《计算机程序的结构和解释(SICP)》Abelson 和 Sussman
- 健壮性：输入非法数据时，能做出对应的处理，而不是产生莫名其妙的错误
- 效率与低存储连需求：执行时间尽可能短、所需要的最大存储空间要尽可能少

### 大O记法

时间复杂度（T(n)）：程序所执行的最大步数
空间复杂度（S(n)）：程序运行过程中临时占用的最大存储空间

**我们使用 大O记法 来标识算法的执行情况：最好情况、平均情况、最坏情况，通常来说我们都是根据「最坏情况」进行分析。（这种悲观主义其实是很有用的：知道各种算法会差到什么程度，能使我们做好最坏打算，以选出最适合的算法。）**

**1、大O记法 解答的是这样的问题：当数据增长时，步数如何变化？ ==> 表示的是一个算法执行效率与数据规模**

==大 O 记法只表明，对于不同分类的算法，存在一**临界点** ，在这一点之后，一类算法会快于另一类，并永远保持下去。至于这个点在哪里，大 O 并不关心。==

2、时间复杂度（T(n)）的几个概念

- 常数时间：O(1) 则为一条水平线，因为不管数据量是多少，算法的步数都恒定。所以，O(1) 也被称为常数时间 。**O(1) 用来表示所有数据增长但步数不变的算法复杂度**。即：O(3) 应该写成 O(1).
- 非常数时间：如 O(N) 呈现为一条对角线。当数据增加一个单位时，算法也随之增加一步。也就是说，数据越多，算法所需的步数就越多。O(N) 也被称为线性时间 。

![大 O 记法](https://cdn.jsdelivr.net/gh/MinsonLee/minsonlee.github.io/images/pig/big-O-in-algorithm.png)

3、时间复杂度的几个计算原则
- **加法原则：总复杂度等于量级最大的那段代码的复杂度**。
    - 1、为什么 O(100) + O(N) = O(N)、O(N) + O(N) = O(N)？
        - O(100) 属于常数时间，与 N 无关，随着 N 趋向于无穷大的时候，常数时间就可以忽略不计，因此：O(100) + O(N) = O(N)。
        - 理论上：O(N) + O(N) = O(2N)，同上理论所得：随着 N 规模的增大，此时就可以忽略与 N 最高次项相乘的常数，因此 O(2N) = O(N)
    - 2、为什么 、O(N^2) + O(N) = O(N^2)？
        - 一般来说，计算机的执行次数都是十分巨大的，因此我们不需要在意倍数关系或者常数的加减关系。我们需要关心的就是存在指数爆炸的指数关系。换言之，我们在得到核心代码执行次数的函数表达之后，只需要找到这一函数的最高次幂，以其为代表进行比较即可。 
- **乘法原则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积**

### 常见算法

#### 二分查找：O(log N)

1. 首先，设定下界和上界，以限定所查之值可能出现的区域。在开始时，以数组的第一个元素为下界，以最后一个元素为上界
2. 循环检查上界和下界之间的最中间的元素（若中间有两个，则取大。即：向上取整）
3. 如此找出最中间的格子之索引，获取该中间格子的值，如果该值正是我们想查的，那就完事了。否则，看你是要往上找还是往下找，来调整下界或上界。
4. 重复上述步骤：当下界超越上界，便知数组里并没有我们所要找的值

#### 冒泡排序 O(N^2) = O(n(n-1)/2) = O(n^2 - n) = O(n^2) - O(1) = O(n^2)

> 每次循环，都将序列中最大的哪一个值“冒泡”到末尾，然后将下次比较的序列长度减 1。

1. 第1次循环：将 arr[1] 和 arr[0] 做比较，若：`arr[0] > arr[1]` 大，则交换两者位置。比较完毕后，继续比较 `arr[1] > arr[2]`...`arr[n-1] > arr[n]`，此时：`arr[n]`就已经是当前序列中最大的值。
2. 第2次循环：比较新数组的 arr[1] 和 arr[0] 做比较，若：`arr[0] > arr[1]` 大，则交换两者位置。比较完毕后，继续比较 `arr[1] > arr[2]`...`arr[n-2] > arr[n-1]`
3. 第 `m` 次循环：比较新数组的 arr[1] 和 arr[0] 做比较，若：`arr[0] > arr[1]` 大，则交换两者位置。比较完毕后，继续比较 `arr[1] > arr[2]`...`arr[n-m] > arr[n-m+1]`

- 等差数列求和：(n-1) + (n-2) + ... + 1
高斯求和：和= (首项 + 末项) *  项数 / 2。

即：(((n-1)+1) * (n-1)) / 2 = (n * (n-1))/2 = (n^2 -n)/2 = (n^2)/2 = n^2 * 1/2 = n^2

```php
// 最坏的情况 O(N^2)：每个元素都需要比较，总计需要 n(n-1)/2 步。
<?php
function bubbleSort($arr)
{
    $len = count($arr);
    for ($i = 0; $i < $len - 1; $i++) 
    {
        for ($j = 1; $j < $len - $i; $j++) 
        {
            if ($arr[$j-1] > $arr[$j]) 
            {
                $arr[$j] += $arr[$j-1];
                $arr[$j-1] = $arr[$j] - $arr[$j-1];
                $arr[$j] -= $arr[$j-1];
            }
        }
    }
    return $arr;
}
```

最好的情况是：完全是正序的数组。若在任意一次循环比较过程中，该过程没有发生过任何一次交换，则意味着当前数组已经是一个排好序的数组，可以提前结束比较。O(n-1) = O(N)

```php
<?php
function bubbleSort($arr)
{
    $len = count($arr);
    for ($i = 0; $i < $len - 1; $i++) 
    {
        $break = true;
        for ($j = 1; $j < $len - $i; $j++) 
        {
            if ($arr[$j-1] > $arr[$j]) 
            {
                $arr[$j] += $arr[$j-1];
                $arr[$j-1] = $arr[$j] - $arr[$j-1];
                $arr[$j] -= $arr[$j-1];
                $break = false;
            }
        }
        
        if ($break === true) break;
    }
    return $arr;
}
```

```php
<?php
// 枚举法-时间复杂度：O(N^2)、空间复杂度：O(1)
class Solution {

    /**
     * @param Integer[] $nums
     * @param Integer $k
     * @return Integer
     */
    function subarraySum($nums, $k) {
        $len = count($nums);
        if ($len < 1) {
            return 0;
        }
        $res = 0;
        for ($i = 0; $i < $len; $i++) {
            $sum = $nums[$i];
            if ($sum == $k) {
                $res++;
            }
            for($j=$i+1; $j < $len; $j++) {
                $sum += $nums[$j];
                if($sum == $k) $res++;
            }
        }
        return $res;
    }
}
```
